# UI画面ワイヤー仕様書

## 1. 目的

本書は、減色ツールが最終的にどんな画面になるのかを、実装前に共通認識化するためのワイヤー仕様書にゃ。

ここでは次を明確にするにゃ。

- PCでどんな画面構成になるか
- スマートフォンでどんな画面構成になるか
- ユーザーがどの順に操作するか
- ROI、BGパレット、スプライト情報がどこに見えるか

## 2. 全体コンセプト

このツールは「画像編集アプリ」というより、`減色結果を見ながら制約を調整する作業台` として設計するにゃ。

そのため、画面の中心は常に `画像キャンバス` にするにゃ。  
設定と結果情報は左右に広げず、上から下へ素直に積んで、気軽に使える縦長ツールとしてまとめるにゃ。

## 3. PC版レイアウト

### 3.1 基本構成

PC版も `1カラムの縦積み` を基本にするにゃ。

```text
+---------------------------------------------------+
| Header                                            |
+---------------------------------------------------+
| Top Control Area                                  |
+---------------------------------------------------+
| Main Canvas Work Area                             |
+---------------------------------------------------+
| Bottom Info Area                                  |
+---------------------------------------------------+
| Footer / Status                                   |
+---------------------------------------------------+
```

### 3.2 ヘッダー

ヘッダーには次を置くにゃ。

- ツール名
- 現在の対象機種
- 現在の表示倍率
- `再計算` ボタン
- `設定を初期化` ボタン

ヘッダーは高くしすぎないにゃ。  
目的は情報表示より、作業状態の確認にゃ。

### 3.3 上部設定エリア

上部設定エリアは「入力と設定」に専念させるにゃ。

推奨セクション構成:

```text
[画像入力]
- 画像を選択
- クリップボードから貼り付け案内

[機種設定]
- ファミコン向けの減色設定
- BG優先 / スプライト優先

[ROI操作]
- ROI有効/無効
- ROIクリア

[表示設定]
- BGのみ
- スプライトのみ
- ROI表示
- 制約違反表示
```

### 3.4 中央キャンバス作業領域

中央カラムは最重要領域にゃ。

ここには次を置くにゃ。

- 元画像または減色後画像を表示する大きなキャンバス
- ROI矩形オーバーレイ
- 選択中ROIのハンドル
- 必要ならタイルグリッド
- 必要なら属性領域グリッド

推奨構成:

```text
+---------------------------------------------------+
| View Tabs: [最終画像] [BG] [Sprite] [差分]        |
+---------------------------------------------------+
|                                                   |
|                  Main Canvas                       |
|      ROI枠 / グリッド / 警告表示を重ねる           |
|                                                   |
|                                                   |
+---------------------------------------------------+
| Zoom -   100%   Zoom +   Fit   1:1                |
+---------------------------------------------------+
```

### 3.5 下部情報エリア

下部情報エリアは「結果の内訳」に専念させるにゃ。

推奨セクション構成:

```text
[BGパレット]
- 使用中パレット一覧

[スプライトパレット]
- 使用中パレット一覧

[スプライト候補]
- 数
- サイズ
- 警告

[ROI統計]
- 重み
- 対象タイル数
- 優先抽出数

[制約警告]
- 走査線超過
- パレット超過
- タイル色数超過
```

### 3.6 フッター / ステータス

画面下部には短いステータスバーを置くにゃ。

- 現在座標
- 現在タイル座標
- ROIサイズ
- 再計算状態

## 4. PC版ワイヤー詳細

### 4.1 初期状態

画像未読込時は、中央キャンバスに大きく案内を出すにゃ。

```text
+--------------------------------------+
|                                      |
|   画像を読み込んでにゃ               |
|   ・ファイル選択                     |
|   ・Ctrl+V で貼り付け                |
|                                      |
+--------------------------------------+
```

### 4.2 画像読込後

画像読込後は、上から `設定 -> キャンバス -> 内訳` の順に縦へ並ぶにゃ。

```text
+-----------------------------------------------+
| 画像入力 / 機種設定 / ROI操作 / 表示設定      |
+-----------------------------------------------+
| [最終][BG][SP][差分]                          |
|                                               |
|              画像キャンバス + ROI枠           |
|                                               |
+-----------------------------------------------+
| BGパレット / SPパレット / ROI統計 / 制約警告  |
+-----------------------------------------------+
```

### 4.3 ROI作成中

画像上をドラッグしてROIを決めるときは、今が作成中であることが分かるようにするにゃ。

```text
Canvas上でドラッグ中:

+-------------------------------+
|                               |
|     +------------------+      |
|     | 作成中ROI        |      |
|     |                  |      |
|     +------------------+      |
|                               |
+-------------------------------+
```

### 4.4 ROI選択中

選択中ROIにはハンドルを出すにゃ。

```text
      o------------------o
      |                  |
      |    選択中ROI     |
      |                  |
      o------------------o
```

PCでは四隅 + 辺中央ハンドルを出してよいにゃ。

## 5. スマホ版レイアウト

### 5.1 基本構成

スマホは `単カラム` にするにゃ。

```text
+---------------------------+
| Header                    |
+---------------------------+
| Image Input               |
+---------------------------+
| Machine Settings          |
+---------------------------+
| ROI Action Buttons        |
+---------------------------+
| Main Canvas               |
+---------------------------+
| View Tabs                 |
+---------------------------+
| BG Palette / SP Palette   |
+---------------------------+
| ROI Stats / Warnings      |
+---------------------------+
```

### 5.2 スマホでの重要方針

- キャンバスを最優先にゃ
- 情報ペインは折りたたむにゃ
- ROI操作は明示的モード切替にするにゃ

PC版も基本思想は同じにゃ。  
違いは、PC版では各セクションの横幅に余裕があるので、設定項目や情報一覧を少し広く見せられる点にあるにゃ。

### 5.3 スマホ用上部構成

```text
[画像を選択]
[ファミコン減色]
[再計算]

[ROI有効] [ROIクリア]
```

### 5.4 スマホ用キャンバス部

```text
+---------------------------+
| [最終] [BG] [SP] [差分]   |
+---------------------------+
|                           |
|        Main Canvas        |
|        ROIオーバーレイ     |
|                           |
+---------------------------+
| [-] 100% [+] [Fit]        |
+---------------------------+
```

### 5.5 スマホ用下部情報

下部情報はアコーディオンかタブにするにゃ。

```text
[BGパレット]
[スプライトパレット]
[ROI統計]
[警告]
```

開いたセクションだけ内容を見せるにゃ。

## 6. 操作フロー

### 6.1 典型的なPC操作フロー

1. 画像を読み込むにゃ
2. 中央キャンバス上をドラッグしてROIを作るにゃ
3. 下部のパレットや警告の変化を見るにゃ
4. 必要ならROIを動かすにゃ
5. 表示タブを切り替えて BG / Sprite の使われ方を確認するにゃ

### 6.2 典型的なスマホ操作フロー

1. 画像を読み込むにゃ
2. キャンバス上でドラッグしてROIを作るにゃ
3. `ROI統計` や `警告` を開いて確認するにゃ
4. 必要ならROIをドラッグして調整するにゃ

## 7. 表示タブ仕様

中央キャンバス上のタブは少なくとも次を持つにゃ。

- `最終画像`
- `BG`
- `Sprite`
- `差分`

### 7.1 最終画像

- BG + Sprite 合成結果を表示するにゃ
- ROIオーバーレイもここで見えるにゃ

### 7.2 BG

- BGだけを表示するにゃ
- パレットやタイル統合の影響を見やすくするにゃ

### 7.3 Sprite

- スプライト候補だけを表示するにゃ
- ROIによってどこが救済されたか確認しやすくするにゃ

### 7.4 差分

- 元画像と最終画像の差を見るにゃ
- ROIの効き具合確認に有効にゃ

## 8. ROI編集UIの見せ方

### 8.1 単一ROI設定

上部設定エリアまたは下部折りたたみ内に、現在のROI状態を置くにゃ。

表示項目は次で十分にゃ。

- 有効/無効
- 重み
- クリアボタン

### 8.2 選択中ROI詳細

選択中ROIに対しては次を出すにゃ。

- X
- Y
- Width
- Height
- Detail Weight

初版では数値入力を持ってもよいけれど、主要操作はあくまでマウス / タッチにゃ。

## 9. 警告表示の見せ方

警告はモーダルでなく、下部情報エリアまたは下部折りたたみで常時確認できるようにするにゃ。

優先表示順:

1. スプライト走査線超過
2. パレット超過
3. タイル色数超過
4. ROI競合

警告行を押すと、該当範囲をキャンバス上で強調できると理想にゃ。

## 10. 初版で割り切ってよい項目

初版では簡略化してよいもの:

- 複数ウィンドウ
- 複雑なドッキングUI
- 自由なペインリサイズ
- デスクトップアプリ風の重いツールバー

ただし、以下は割り切ってはいけないにゃ。

- 単カラムでも中央に大きなキャンバスがあること
- ROIがキャンバス上で直接編集できること
- BGパレットとスプライト情報が同時に見えること
- PCとスマホのどちらでも単カラムで破綻なく使えること

## 11. 実装チェックリスト

以下を満たせば、画面構成は初版として成立しているにゃ。

1. PCで `上:設定 / 中央:キャンバス / 下:情報` の単カラムになっているにゃ
2. スマホでも同じ構造で単カラムのまま使えるにゃ
3. 画像未読込時の案内が分かりやすいにゃ
4. ROIをキャンバス上で直接編集できるにゃ
5. BGパレットとスプライト情報が同時に確認できるにゃ
6. `最終 / BG / Sprite / 差分` の表示切替ができるにゃ
7. 警告が別画面に隠れず確認できるにゃ
