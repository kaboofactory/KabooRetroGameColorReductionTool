# ROI操作仕様書

## 1. 目的

本書は、減色ツールにおける `ROI (Region of Interest)` の操作仕様を定義するにゃ。

ここでいうROIは、ユーザーが矩形で指定する「ディテール優先範囲」を指すにゃ。  
この範囲では、BGパレット統合よりも細部保持を優先し、必要に応じてスプライト候補への再配分を強めるにゃ。

本仕様書の目的は次のとおりにゃ。

- ROIの入力方式を統一するにゃ
- マウスだけで主要操作が完結するようにするにゃ
- スマートフォンでも同等の操作ができるようにするにゃ
- 実装時に入力状態や誤操作対策で迷わないようにするにゃ
- ROIは1つだけに限定して、操作と実装をシンプルに保つにゃ

## 2. 前提

### 2.1 操作原則

- 主要操作はマウスだけで完結するにゃ
- タブレット / スマートフォンでは同等の操作をタッチで提供するにゃ
- キーボードは補助にゃ
- ROI操作は画像キャンバス上で完結するにゃ

### 2.2 ROIの形

- 初版では `軸に平行な矩形` のみを扱うにゃ
- 回転矩形、自由形状、多角形ROIは対象外にゃ

### 2.3 ROIの用途

ROIは表示用マークではなく、減色アルゴリズムへ影響するにゃ。

- ROI内タイルの誤差重みを上げるにゃ
- ROI内ではBGで無理にまとめず、スプライト候補へ逃がしやすくするにゃ
- 初版ではROIは1つだけ持ち、その1領域に対して重みを適用するにゃ

## 3. ROIデータモデル

初版では少なくとも次を持つにゃ。

```js
type RegionOfInterest = {
  x: number;
  y: number;
  width: number;
  height: number;
  detailWeight: number;
  enabled: boolean;
};
```

### 3.1 各項目の意味

- `x`, `y`: 画像座標系での左上位置にゃ
- `width`, `height`: 画像座標系でのサイズにゃ
- `detailWeight`: ディテール優先度の重みにゃ
- `enabled`: 一時無効化用フラグにゃ

### 3.2 座標系

- ROI座標は `元画像基準` または `処理対象キャンバス基準` の論理座標で保持するにゃ
- CSSピクセルや表示倍率依存の座標をそのまま保存してはいけないにゃ
- ズームやレスポンシブ縮小があっても、ROI自体は同じ画像上の範囲を指し続ける必要があるにゃ

## 4. 入力状態モデル

操作系は少なくとも次の状態を持つにゃ。

```js
type InputMode =
  | "idle"
  | "creating-roi"
  | "moving-roi"
  | "resizing-roi"
  | "panning-image";

type InputState = {
  mode: InputMode;
  activePointerId: number | null;
  activeHandle: string | null;
  dragStartImageX: number;
  dragStartImageY: number;
  lastImageX: number;
  lastImageY: number;
};
```

### 4.1 初版で必要な状態

- `idle`: 待機中にゃ
- `creating-roi`: 新規ROI作成ドラッグ中にゃ
- `moving-roi`: ROI本体移動中にゃ
- `resizing-roi`: ROIハンドル操作中にゃ
- `panning-image`: 将来の画像パン操作用に予約しておくにゃ

### 4.2 同時入力

- 初版では `1 pointer only` にするにゃ
- マルチタッチのピンチズームは将来拡張にゃ

## 5. ROIの基本操作

### 5.1 新規作成

PC:

- 画像キャンバス上でマウスダウンするにゃ
- そのままドラッグして矩形を広げるにゃ
- マウスアップで確定するにゃ
- 既存ROIがあれば、新しいドラッグ結果で置き換えるにゃ

スマホ:

- 画像キャンバス上でタッチ開始するにゃ
- ドラッグして矩形を広げるにゃ
- 指を離して確定するにゃ
- 既存ROIがあれば、新しいドラッグ結果で置き換えるにゃ

### 5.2 選択

PC / スマホ共通:

- ROI本体をタップ / クリックすると選択するにゃ
- 選択中ROIは枠色やハンドル表示で明確に見分けられるようにするにゃ

### 5.3 移動

PC:

- 選択中ROIの内側をドラッグすると移動するにゃ

スマホ:

- 選択中ROIの内側をドラッグすると移動するにゃ
- 誤操作防止のため、短時間タップとドラッグ開始を分けて判定するにゃ

### 5.4 リサイズ

PC:

- ROI四隅と四辺中央のハンドルをドラッグしてリサイズするにゃ

スマホ:

- 四隅ハンドルのみを初版対象にするにゃ
- 辺中央ハンドルは画面が狭いと誤操作しやすいため、初版では省略してよいにゃ

### 5.5 一時無効化

削除の代わりに、一時的に無効化できると便利にゃ。

- 上部設定から `有効/無効` を切り替えるにゃ
- 無効ROIは画面上では薄い色で表示するにゃ
- 減色計算には影響しないにゃ

## 6. ROI作成ルール

### 6.1 最小サイズ

- 幅または高さが一定未満の矩形は作成しないにゃ
- 初版では `4px` または `1 tile` 相当を下限候補にするにゃ

### 6.2 画面外制約

- ROIは画像範囲外へはみ出して保持しないにゃ
- 作成中や移動中に外へ出そうになったら、画像境界でクランプするにゃ

### 6.3 正規化

ドラッグ方向は自由にゃ。

- 左上から右下
- 右下から左上
- 左下から右上
- 右上から左下

どの方向でも、確定時には `x, y, width, height` を正規化するにゃ。

## 7. ROI選択ルール

### 7.1 ヒットテスト優先順位

ROIは1つだけなので、ヒットテストは次で十分にゃ。

1. リサイズハンドル
2. ROI本体

## 8. マウス操作仕様

### 8.1 入力イベント方針

初版では `Pointer Events` を使って統一実装するのが望ましいにゃ。

理由:

- マウスとタッチを1系統で扱いやすいにゃ
- 外部ライブラリ不要にゃ

### 8.2 新規ROI作成フロー

1. `pointerdown` で開始点を保存するにゃ
2. `pointermove` で仮矩形を更新するにゃ
3. `pointerup` で矩形を確定するにゃ
4. 最小サイズ未満なら破棄するにゃ

### 8.3 ROI移動フロー

1. ROI本体上で `pointerdown` するにゃ
2. ROI選択状態を確定するにゃ
3. 移動量に応じて `x`, `y` を更新するにゃ
4. 画像境界でクランプするにゃ
5. `pointerup` で確定するにゃ

### 8.4 ROIリサイズフロー

1. ハンドル上で `pointerdown` するにゃ
2. 対応辺だけを更新するにゃ
3. 反転した場合は矩形を正規化するにゃ
4. 最小サイズ未満は補正するにゃ
5. `pointerup` で確定するにゃ

## 9. タッチ操作仕様

### 9.1 基本方針

- マウスと同じ操作概念を使うにゃ
- ただし誤操作を避けるため、ヒット領域は広くするにゃ

### 9.2 タッチターゲット

- ハンドルは見た目より広い当たり判定を持たせるにゃ
- 初版は `最低 32x32 CSS px` 程度を目安にするにゃ

### 9.3 誤スクロール防止

キャンバス上でROI操作中は、ページスクロールと競合しやすいにゃ。

方針:

- ROI操作領域では必要に応じて `touch-action: none` を使うにゃ
- ただしページ全体のスクロール不能にはしないにゃ
- キャンバス領域だけに限定するにゃ

### 9.4 スマホ時の追加UI

スマホでは、常時ハンドルを小さく出すより次が安全にゃ。

- ROI選択時のみハンドルを表示するにゃ
- 選択中ROIに `移動` と `削除` の補助ボタンを近接表示してもよいにゃ

## 10. スマートフォン対応レイアウト

### 10.1 基本方針

- PCは作業キャンバス中心の複数ペイン構成にゃ
- スマホは単カラム構成にゃ

### 10.2 スマホ時の並び

推奨順:

1. 画像入力
2. 機種選択
3. ROI操作ボタン群
4. キャンバス
5. 表示切替
6. BGパレット / スプライトパレット
7. ROI統計
8. 警告一覧

### 10.3 スマホ時のROI操作ボタン

少なくとも次を置くにゃ。

- `ROI有効/無効`
- `ROIクリア`

初版では、キャンバス上のドラッグで直接ROIを決める方式を優先するにゃ。

## 11. ROIと減色アルゴリズムの接続

### 11.1 ROI重み

ROIは1つだけ持ち、初版では `1.0` を基準、強調時は `2.0` や `3.0` のように重くするにゃ。

### 11.2 タイル重みへの変換

各タイルについて:

- ROIと重ならない: 重み `1.0`
- 一部重なる: 重なり率に応じて補間
- 完全に含まれる: ROI重みを最大適用

### 11.3 スプライト候補への影響

ROI内では:

- BGへ押し込むコストを上げる
- スプライト候補化の閾値を下げる

これにより、細部を残したい領域をスプライト側で救いやすくするにゃ。

## 12. エラーハンドリング

### 12.1 作成失敗

- 最小サイズ未満なら作成しないにゃ
- 画像未読込ならROIを作成できないにゃ

### 12.2 編集失敗

- 無効ROIは移動できないようにしてもよいにゃ
- 画像範囲外へは移動させないにゃ

## 13. 初版で割り切ってよい項目

初版では簡略化してよいもの:

- 回転ROI
- 多角形ROI
- マルチタッチピンチズーム
- ROIの複数管理

ただし、以下は割り切ってはいけないにゃ。

- マウスだけでROI作成、移動、無効化まで完結すること
- タッチでも同等操作ができること
- ROI座標が画像基準で保存されること
- スマホでROI編集が破綻しないこと
- ROIが実際に減色重みに影響すること

## 14. 実装チェックリスト

以下を満たせば、ROI操作仕様は初版として成立しているにゃ。

1. 画像上をドラッグしてROIを作成できるにゃ
2. ROIを選択できるにゃ
3. ROIを移動できるにゃ
4. ROIをリサイズできるにゃ
5. ROIをクリアまたは無効化できるにゃ
6. マウス操作だけで主要操作が完結するにゃ
7. タッチ操作でも主要操作が完結するにゃ
8. スマホでハンドル誤操作が抑制されているにゃ
9. ROIが画像外へはみ出さないにゃ
10. ROI重みがタイル解析と減色に反映されるにゃ
